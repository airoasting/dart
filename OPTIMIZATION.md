# dart 스킬 개선 기록 (2026-07-04)

이 문서는 dart 스킬을 하루에 걸쳐 개선한 전체 내역이다. 문제 → 해결 → 효과 순으로 정리한다.

## 한 줄 요약

리포트 한 건에 **10분**이 걸리던 스킬을 **3~4분**으로 줄이고, 주가·기준일·차트의 정확도 결함을 잡고, 유지보수 가능한 구조(단일 데이터 계약)로 바꿨다.

## 1. 템플릿 + 빌더 아키텍처 (핵심)

**문제.** LLM이 매번 1,400줄 HTML을 처음부터 토큰 단위로 다시 뱉었다. 그중 ~1,200줄(CSS·차트 코드·레이아웃)은 모든 리포트에서 동일한 보일러플레이트였다. 출력 토큰의 90%가 낭비였고 이게 10분의 정체였다.

**해결.** 보일러플레이트를 `assets/template.html`로 고정하고, LLM은 데이터(`data.json`)만 만든다. `assets/build_report.py`가 둘을 결합한다(결정론적, 1초).

**효과.** LLM 출력 1,400줄 → data.json ~200줄. HTML 생성 병목이 사라졌다.

## 2. 2차 병목 — 병렬화

**문제.** HTML 생성이 1초로 줄자 웹 리서치·서술 생성이 직렬이라 여전히 느렸다(5~8분).

**해결.** 재무 숫자만 나오면 나머지는 독립적이다. `Step 4.5`에서 서브에이전트 3개(웹 리서치 / 13 페르소나 / Bull·Bear)를 **동시에** 띄운다.

**효과.** 이 구간 5~8분 → 2~3분. 실측: 카카오 A 115초·B 72초·C 33초를 병렬로 ~115초.

## 3. 정확도 결함 교정

- **차트 Y축 하드코딩 4곳 제거.** 델타(-1000), 목표가(40000/90000), 주가 라인(44000/51000), 세그먼트(자동 max)가 데이터에 맞지 않아 타 종목·큰 값에서 라벨이 잘리거나 축이 어긋났다(kakao는 값이 우연히 맞아 안 드러남). 전부 데이터 기반 20~30% 여유 + 상단 패딩으로 교정. NAVER(20만원대)·SK하이닉스(52조)에서 발견·검증.
- **주가는 전일 종가로 고정.** 웹 검색 현재가가 출처마다 달라 부정확(카카오 실측 대비 10% 오차). `assets/price.py`(Naver 금융 KRX 일봉, requests만 필요)로 전일 종가·등락·52주 범위를 결정론적으로 가져온다.
- **기준일은 실행일로.** `meta.base_date`를 빌더가 항상 빌드 실행일(오늘)로 자동 스탬프한다. 주가(전일 종가)와 기준일(생성일)을 분리.
- **buildMarket 일반화.** 애널리스트 배지가 항상 badge-buy로 고정되던 버그, 도넛 수·뉴스검색 종목명 하드코딩을 데이터 기반으로 교정.

## 4. 구조·UI 정리

- **주가 반응 섹션(sec-stock) 제거.** 30영업일 주가 재구성이 웹 검색을 가장 많이 잡아먹고 부정확해 삭제. 애널리스트 목표가는 sec-market에 그대로. 7 → 6개 섹션.
- **왼쪽 색 띠 제거.** 섹션 라벨 앞 코랄 세로 바(`.sec-label::before`) 삭제.
- **사업별 매출에 "단위: 억원" 표기 추가.**
- **페르소나 13파일 → `_ALL.md` 단일 읽기**, `design-system.md`·`section-templates.md`는 런타임 미참조(template.html에 반영됨).
- **저장소 재배치.** `dart/` 하위 폴더 → 프로젝트 루트로 이동(루트가 곧 airoasting/dart 저장소).

## 5. 유지보수 장치

- **단일 데이터 계약.** `assets/data.example.json`이 스키마이자 예시. LLM은 이 구조만 채운다.
- **빌더 검증.** 미치환 토큰·데이터 누락·페르소나 13인·Chart.js 4.4.4를 확인해 실패 시 중단.
- **회귀 가드.** 템플릿 생성 스크립트가 하드코딩(`min:-1000`·`min:44000`·`sec-stock` 등) 재유입 시 assert로 실패.

## 효과 정리

| 항목 | 이전 | 이후 |
|------|------|------|
| 리포트 생성 시간 | 약 10분 | **3~4분** (병렬 웹 리서치가 하한) |
| LLM 출력량 | HTML 1,400줄 | data.json ~200줄 |
| 주가 | 웹 검색(부정확, ~10% 오차) | KRX 전일 종가 |
| 차트 축 | 일부 하드코딩(타사 오류) | 전부 데이터 자동 계산 |
| 디자인 일관성 | 매번 재생성 → 흔들림 | 템플릿 고정 → 픽셀 동일 |

**검증 종목.** 카카오(1.9조)·NAVER(3.2조)·SK하이닉스(52조) 세 스케일에서 렌더·데이터 정합·차트 축을 확인했다.

## 재사용 원리

"LLM이 큰 산출물을 통째로 생성하는데 대부분이 고정 구조"라면 어디든 같은 처방이 먹힌다 — **구조는 템플릿, LLM은 데이터, 결합은 스크립트.** 그리고 값이 변하는 지표(주가 등)는 웹 검색이 아니라 결정론적 소스에서 가져온다. news-research·steel-brief·web-builder처럼 HTML을 뱉는 다른 스킬도 후보다.

```
assets/

  data.example.json    데이터 계약(스키마) + 카카오 예시
  price.py             전일 종가·52주 (KRX)
  dart_client.py       DART Open API 클라이언트
investor_persona/_ALL.md   13인 페르소나 통합본
```
