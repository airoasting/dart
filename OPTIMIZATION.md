# dart 스킬 분석 및 최적화 (2026-07-04)

## 왜 (문제)

리포트 한 건에 약 10분이 걸렸다. 원인은 하나였다.

**LLM이 매번 1,441줄짜리 HTML을 처음부터 토큰 단위로 다시 뱉었다.** 그중 약 1,200줄(CSS 523줄 + 차트·레이아웃 JS 약 700줄)은 카카오든 네이버든 **모든 리포트에서 글자 하나 안 바뀌는 보일러플레이트**였다. 기업마다 실제로 달라지는 건 데이터(재무 숫자·세그먼트·주가·뉴스·페르소나 평가)뿐인데, 스킬은 그 데이터를 채우려고 보일러플레이트 전체를 매번 재생산하고 있었다.

출력 토큰 약 70,000개 중 90%가 낭비였다. 이게 10분의 정체다.

부수 문제도 있었다.

- **설치본이 낡아 있었다.** 실제로 돌아가는 `~/.claude/skills/dart/SKILL.md`는 78KB짜리 구버전 모놀리식(CSS/HTML/JS 전부 인라인)이었고, `references/`로 분리한 신버전은 프로젝트 폴더에만 있고 배포되지 않았다.
- **정답 HTML 자체에 하드코딩이 있었다.** 스킬은 "Y축 하드코딩 절대 금지"를 규칙으로 내걸었지만, ground-truth인 `kakao_20260510.html`의 델타 차트(`min:-1000`)와 목표가 차트(`min:40000,max:90000`)는 하드코딩이었다. 다른 기업엔 그대로 틀린 값이 된다.
- **버그.** `buildMarket`의 애널리스트 배지가 항상 `badge-buy`로 고정(카카오는 6곳 다 Buy라 우연히 안 드러남), 컨센 도넛 `[21,6,0]`·뉴스검색 종목명 `'카카오'`·목표가 라벨이 함수 안에 하드코딩돼 있었다.

## 무엇 (해결)

**"보일러플레이트 고정 + 데이터 주입" 아키텍처로 전환했다.**

```
과거:  LLM ─────────────────────────▶ HTML 1,441줄 (매번 재생성)

현재:  template.html (고정 1,200줄)  ┐
       data.json (LLM이 쓴 ~200줄)   ├─▶ build_report.py ─▶ HTML (1초)
                                     ┘
```

- **`assets/template.html`** — 정답 HTML에서 데이터만 걷어내고 보일러플레이트만 남긴 고정 템플릿. 데이터 자리엔 `/*__REPORT_DATA__*/` 주입 지점과 `{{토큰}}` 54개.
- **`assets/build_report.py`** — `data.json`을 받아 `const` 선언 주입 + 토큰 치환 + 필터 버튼 생성 + 검증(미치환 토큰·페르소나 13인·Chart.js 4.4.4) 후 `output/`에 저장.
- **`assets/data.example.json`** — 데이터 계약이자 카카오 실제 예시. LLM은 이 구조를 복사해 값만 바꾼다.
- **`SKILL.md`** — "HTML을 쓰지 마라. data.json만 만들고 빌더를 돌려라"로 재작성.
- **`investor_persona/_ALL.md`** — 13개 페르소나 파일을 1개로 합쳐 파일 읽기 13회 → 1회.

같이 고친 것:
- 델타·목표가 차트 Y축 하드코딩 제거 → 데이터 기반 자동 계산(목표가 축은 기존 40000/90000을 정확히 재현, 델타 축은 데이터에 맞게 교정).
- 배지 계산 버그 수정, 도넛·종목명·목표가 라벨을 데이터 기반으로 일반화.
- 낡은 설치본을 신버전으로 배포.

## 효과 (검증)

| 항목 | 과거 | 현재 |
|------|------|------|
| LLM 출력량 | HTML 1,441줄 (~70K 토큰) | data.json ~200줄 (~7K 토큰) |
| 리포트 생성 시간 | 약 10분 | **2분 안쪽** (이제 DART API·WebSearch 수집이 하한) |
| 런타임 파일 읽기 | SKILL.md(78KB) + 페르소나 13파일 | data.example.json + 페르소나 1파일 |
| 차트 축 | 일부 하드코딩(타사엔 오류) | 전부 데이터 자동 계산 |
| 디자인 일관성 | 매번 재생성 → 흔들림 위험 | 템플릿 고정 → 픽셀 동일 보장 |

**회귀 테스트**: 카카오 데이터(`data.example.json`)로 빌드한 결과를 정답과 대조 →
CSS 바이트 동일, 함수부는 의도한 9개 일반화만 차이, 전 데이터 배열 개수·값 일치, 브라우저 렌더 시 5개 차트 정상·JS 에러 0·미치환 토큰 0.

## 남은 기회 (다음 단계)

1. **네이버 예시 회귀 테스트 추가.** `네이버_20260510.html`도 data.json으로 역추출해 두 번째 골든 케이스로 두면, 템플릿 수정 시 회귀를 2건으로 잡는다.
2. **페르소나 평가 병렬화.** 13인 평가는 여전히 LLM 추론이다. 서브에이전트로 병렬 실행하면 이 구간도 단축 가능(현재는 데이터 수집이 하한이라 우선순위 낮음).
3. **data.json 스키마 파일.** `data.example.json` 옆에 JSON Schema를 두면 빌더가 형식 오류를 더 일찍 잡는다.
4. **CI 훅.** `template.html` 커밋 시 `data.example.json`으로 빌드가 통과하는지 pre-commit에서 자동 확인.

## 원리 (재사용 포인트)

이 패턴은 dart 전용이 아니다. **"LLM이 매번 큰 산출물을 통째로 생성하는데 그중 대부분이 고정 구조"라면 어디든 같은 처방이 먹힌다** — 구조를 템플릿으로 고정하고, LLM은 데이터만, 결합은 결정론적 스크립트로. news-research·steel-brief·web-builder처럼 HTML을 뱉는 다른 스킬도 후보다.
